★★☆ Intermediate — 5 Steps

Your frontend application container can start, but it fails to connect to the backend API container. Both are supposed to be on the same Docker network. Your task is to diagnose why they cannot communicate. This involves checking Docker network configurations, DNS resolution, and container-to-container reachability.

What You’ll Do
  • ▸ Inspect Docker networks with docker network inspect
  • ▸ Test connectivity from one container to another using docker exec, ping, or curl
  • ▸ Verify port mappings and service names used in application configuration
  • ▸ Examine EXPOSE directives in Dockerfile and docker-compose.yml network definitions
